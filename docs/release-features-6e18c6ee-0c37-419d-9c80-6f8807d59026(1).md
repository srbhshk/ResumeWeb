# Feature Strategy

## Overview

- **Total features:** 11

## Core Features

### Resume PDF Upload

- **ID:** `feature-1`
- **Priority:** high
- **Category:** core
- **Description:** Allow users to upload a single resume PDF without requiring signup.
- **User impact:** Users can easily upload their resume without friction or account creation.

**Requirements:**

- Provide a single landing page for uploading one resume PDF.
- Accept only PDF files with MIME type application/pdf.
- Enforce a maximum PDF size of 4MB.
- No user account or login required.

**Technical requirements:**

- Support asynchronous upload handling.
- Validate file type and size before processing.

### Asynchronous Resume Processing with Status

- **ID:** `feature-2`
- **Priority:** high
- **Category:** core
- **Description:** Process uploaded PDF asynchronously to extract structured portfolio data with visible processing status.
- **User impact:** Users receive feedback on processing progress and errors, ensuring transparency and trust.

**Requirements:**

- Parse PDF asynchronously after upload to extract summary, experience, skills, education, and projects.
- Show visible processing status on UI using polling (no long blocking requests).
- Return clear error messages if parsing fails.
- Reject portfolio publishing if parsing fails.

**Technical requirements:**

- Support async processing pipeline with status updates.
- Handle parse failures gracefully with user feedback.

### Public Portfolio Page Generation

- **ID:** `feature-3`
- **Priority:** high
- **Category:** core
- **Description:** Generate a polished, mobile-friendly public portfolio page at a unique URL after successful parsing.
- **User impact:** Users get a shareable, professional portfolio page without exposing private contact details.

**Requirements:**

- Publish portfolio at URL pattern /p/{slug}.
- Strip email, phone, and address from public page at parse time to protect privacy.
- Return 404 when portfolio is unpublished.
- Ensure portfolio is mobile-friendly and visually polished.

**Technical requirements:**

- Generate unique slugs for portfolio URLs.
- Enforce privacy by removing PII from public pages.

### Private Edit Link with Signed Token

- **ID:** `feature-4`
- **Priority:** high
- **Category:** core
- **Description:** Provide users a private edit URL secured by a signed token to update portfolio content without login.
- **User impact:** Users can easily refine and manage their portfolio securely without creating accounts.

**Requirements:**

- Return a private edit link after successful upload and parse.
- Allow editing of text fields extracted from resume.
- Allow optional photo upload to enhance portfolio.
- Allow unpublishing or deleting the portfolio via edit link.
- No user login or account required; access controlled solely by signed token.

**Technical requirements:**

- Implement secure token-based access for editing.
- Support editing UI for text and photo updates.

### SEO and Visibility Controls

- **ID:** `feature-5`
- **Priority:** high
- **Category:** core
- **Description:** Allow portfolio owners to control search engine indexing and visibility of their portfolio pages.
- **User impact:** Users control whether their portfolio is discoverable by search engines, protecting privacy or enabling visibility.

**Requirements:**

- By default, set public portfolio pages to noindex via robots meta tag and HTTP header.
- Provide toggle in edit UI to allow search engines to index the portfolio.
- Implement two flags: is_published (controls 404 visibility) and allow_search_indexing (controls SEO).

**Technical requirements:**

- Support dynamic meta tags and headers based on flags.
- Enforce 404 response when is_published is false.

## Infrastructure Features

### SaaS Web Deployment

- **ID:** `infra-1`
- **Priority:** high
- **Category:** infrastructure
- **Description:** Deploy the application as a SaaS web app accessible via browser without signup.
- **User impact:** Users access the service easily via web without installation or signup.

**Requirements:**

- Support single monolith deployment combining UI and API route handlers.
- Support asynchronous backend processing pipelines.
- Support storage for uploaded PDFs and optional photos.
- Support a small relational database (~3 tables) for portfolio data and tokens.

## Non-Functional Requirements

### Upload Rate Limiting

- **ID:** `nfr-1`
- **Priority:** high
- **Category:** nfr
- **Description:** Limit the number of uploads per IP address to prevent abuse.
- **User impact:** Prevents service abuse and ensures fair usage for all users.

**Requirements:**

- Enforce a maximum of 5 uploads per hour per IP address.

### File Size and MIME Validation

- **ID:** `nfr-2`
- **Priority:** high
- **Category:** nfr
- **Description:** Validate uploaded files to ensure they are PDFs and within size limits.
- **User impact:** Users receive immediate feedback on invalid uploads, improving experience and security.

**Requirements:**

- Reject files larger than 4MB.
- Accept only files with MIME type application/pdf.

### Privacy by Default

- **ID:** `nfr-3`
- **Priority:** high
- **Category:** nfr
- **Description:** Ensure user privacy by stripping contact details from public pages and defaulting to noindex.
- **User impact:** Users' private contact information is protected by default, increasing trust.

**Requirements:**

- Remove email, phone, and address from public portfolio pages at parse time.
- Default portfolio pages to noindex for search engines.

### Error Handling and User Feedback

- **ID:** `nfr-4`
- **Priority:** high
- **Category:** nfr
- **Description:** Provide clear error messages and UX feedback when parsing or publishing fails.
- **User impact:** Users understand issues and can correct them, improving satisfaction.

**Requirements:**

- Show clear error messages if PDF parsing fails.
- Prevent publishing of portfolios if parsing is unsuccessful.

### Performance - Fast End-to-End Processing

- **ID:** `nfr-5`
- **Priority:** high
- **Category:** nfr
- **Description:** Ensure median processing time from upload to live URL is under 2 minutes.
- **User impact:** Users receive quick results, enhancing usability and engagement.

**Requirements:**

- Complete upload, parse, and portfolio generation in less than 2 minutes median time.
